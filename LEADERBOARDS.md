# Лидерборды — конфигурация для консоли Яндекс.Игр

Справочник для настройки лидербордов в личном кабинете. Технические имена в консоли
должны **точно** (регистр важен) совпадать с идентификаторами в коде, иначе отправка
очков молча уходит «в никуда».

- Источник правды в коде: `src/game/leaderboard.ts` (`LB_ID`, `SCORE_BASE`,
  `LB_TOTAL_SCORE`, `LB_JOURNEY_STARS`, `submitBestScore`), `src/state/leaderboardSync.ts`
  и `src/game/scenes/UIScene.ts` (отправка результата).
- У Яндекса **техническое имя лидерборда неизменяемо** — «переименование» = создание
  новой доски; очки старой не переносятся.

---

## 1. Реализовано в коде

### 1a. 16 таблиц по режимам и сложностям (`<режим><Сложность>`)

Отправляются автоматически после каждой победы (`UIScene`), **только если результат
строго лучше** прежнего.

| Режим | easy | medium | hard | expert |
|---|---|---|---|---|
| Классика (`classic`) | `classicEasy` | `classicMedium` | `classicHard` | `classicExpert` |
| На время (`timeAttack`) | `timeAttackEasy` | `timeAttackMedium` | `timeAttackHard` | `timeAttackExpert` |
| Выживание (`survival`) | `survivalEasy` | `survivalMedium` | `survivalHard` | `survivalExpert` |
| Без ошибок (`noMistakes`) | `noMistakesEasy` | `noMistakesMedium` | `noMistakesHard` | `noMistakesExpert` |

> Ранее классика называлась `matching*` — **переименовано в `classic*`** (2026-07-15).

**Метрика.** Время прохождения. Отправляется как **сырые миллисекунды**
(`секунды × 1000`) на доску типа **`time`** с сортировкой **по возрастанию** (меньше
время — выше место). Пример: 28 сек → `28000`. Внутри игры показывается как `M:SS`
(собственная таблица), на поверхностях Яндекса время форматируется нативно.
Прежний трюк `9999 − сек` (`SCORE_BASE`) убран.

### 1b. Агрегатные (standalone) доски — вариант C

Монотонные метрики, отправляются как есть (без инверсии). Пушатся при загрузке (seed
существующего прогресса), после каждой победы и после уровня «Путешествия»
(`syncProgressLeaderboards()`), только если строго лучше.

| Тех. имя | Метрика | Отправка |
|---|---|---|
| `totalScore` | Совокупный **опыт (XP)** игрока (`progress.stats.xp`) | boot + после победы |
| `journeyStars` | **Сумма звёзд** кампании (`totalStars(campaign)`) | boot + после уровня «Путешествия» |

> «Общий счёт» намеренно считается по **XP**, а не по жемчугу: жемчуг — расходуемый
> баланс, покупки опускали бы игрока в рейтинге. XP монотонен.

**Названия для консоли:**

- `totalScore` — ru «Общий счёт» · en «Total Score» · tr «Toplam Skor» · es «Puntuación total» · pt «Pontuación total».
- `journeyStars` — ru «Звёзды в Путешествии» · en «Journey Stars» · tr «Yolculuk Yıldızları» · es «Estrellas de Aventura» · pt «Estrelas da Aventura».

### Настройки в консоли — ДВЕ группы

**16 досок «режим × сложность» (`classic*`, `timeAttack*`, `survival*`, `noMistakes*`):**

| Параметр | Значение |
|---|---|
| Тип лидерборда | **`time`** (миллисекунды) |
| Направление сортировки | **Сортировка по возрастанию** (меньше время — выше) |
| Размер десятичной части счёта | 0 |
| Основной | нет |

**2 агрегатные доски (`totalScore`, `journeyStars`):**

| Параметр | Значение |
|---|---|
| Тип лидерборда | **`numeric`** |
| Направление сортировки | **Сортировка по убыванию** (больше — выше) |
| Размер десятичной части счёта | 0 |
| Основной | **`totalScore` — ДА**; `journeyStars` — нет |

## 2. Предложено, но ПОКА не реализовано (требуют доработки кода)

Эти доски код **не заполняет**. Заводить в консоли имеет смысл только вместе с
доработкой отправки.

### 2.1 «Лучшее время» — `bestTimeClassic`
- **Консоль (предложено):** type **time**, по возрастанию.
- **Названия:** ru «Лучшее время» · en «Best Time» · tr «En İyi Süre» · es «Mejor tiempo» · pt «Melhor tempo».
- **Гэп:** время классики уже шлётся, но **по сложностям** (`classicEasy…Expert`) и как
  numeric-инверсия. Тип `time`+asc конфликтует с этой схемой — нужно выбрать одну. Плюс
  «через все сложности» несравнимо (easy 6 пар vs expert 14).

### 2.2 «Рекорд: На время» — `timeAttackRecord`
- **Консоль:** numeric, по убыванию, 0 знаков.
- **Названия:** ru «Рекорд: На время» · en «Time Attack Record» · tr «Zamana Karşı Rekoru» · es «Récord contrarreloj» · pt «Recorde contra o tempo».
- **Гэп:** сейчас для timeAttack шлётся **время**, а не очки/пары. Нужно определить и слать
  метрику (набранные пары/очки).

### 2.3 «Рекорд: Выживание» — `survivalRecord`
- **Консоль:** numeric, по убыванию, 0 знаков.
- **Названия:** ru «Рекорд: Выживание» · en «Survival Record» · tr «Hayatta Kalma Rekoru» · es «Récord de supervivencia» · pt «Recorde de sobrevivência».
- **Гэп:** сейчас для survival шлётся **время**, а не максимум пар. Нужно слать число пар
  без ошибки.

## 3. Что завести в консоли прямо сейчас

1. 16 досок `classic*/timeAttack*/survival*/noMistakes*` × `Easy/Medium/Hard/Expert`.
2. `totalScore` и `journeyStars`.

Типы: 16 «режим × сложность» — **`time`, сортировка по возрастанию**;
`totalScore`/`journeyStars` — **`numeric`, сортировка по убыванию**; десятичная часть 0
везде; основной — `totalScore`. Три доски из §2 (`bestTimeClassic`, `timeAttackRecord`,
`survivalRecord`) — отдельной задачей вместе с доработкой кода отправки.
